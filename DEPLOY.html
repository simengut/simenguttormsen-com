Deploying simenguttormsen.com

Two paths: GitHub Pages (simpler) or Cloudflare Pages (faster CDN). Pick one. Don't mix.

Recommended: GitHub Pages

Free. Zero config. Auto-renewed Let's Encrypt SSL. You keep DNS at GoDaddy. Total time: ~15 min + DNS propagation (~1–24 h).

Step 1 — Create the GitHub repo

You can either let me run it (you have gh authed as simengut), or do it yourself in the GitHub web UI.

Option A — with the gh CLI

cd "/Users/simenguttormsen/Library/CloudStorage/Dropbox/SimenPhD/website"
gh repo create simengut/simenguttormsen-com --public --source=. --remote=origin --push

Option B — in the browser

  1. Go to github.com/new
  2. Repository name: simenguttormsen-com (or any name you like)
  3. Visibility: Public (required for free Pages)
  4. Do not add README/license/.gitignore — the local repo already has them
  5. Create the repo, then push from terminal: 
    cd "/Users/simenguttormsen/Library/CloudStorage/Dropbox/SimenPhD/website"
git remote add origin git@github.com:simengut/simenguttormsen-com.git
git branch -M main
git push -u origin main

Step 2 — Enable GitHub Pages

  1. In the repo on github.com: Settings → Pages
  2. Source: Deploy from a branch
  3. Branch: main — folder: / (root) — save
  4. Within ~30 seconds the site goes live at https://simengut.github.io/simenguttormsen-com/
  5. In the same Pages settings, set Custom domain to simenguttormsen.com and save. (The CNAME file is already in your repo.)
  6. Tick Enforce HTTPS once available (~5–10 min after DNS resolves).

Step 3 — Configure DNS in GoDaddy

Log into GoDaddy → My Products → simenguttormsen.com → DNS. Delete the default parked A record, then add these:

TypeHost / NamePoints to / ValueTTL
A@185.199.108.153600
A@185.199.109.153600
A@185.199.110.153600
A@185.199.111.153600
CNAMEwwwsimengut.github.io.3600

The four A records are GitHub Pages' published anycast IPs. The CNAME on www sends visitors typing www.simenguttormsen.com to the same place. The trailing dot on the CNAME value is correct — GoDaddy may strip or add it automatically.

Watch out: GoDaddy's DNS panel often has a default CNAME www → @ record. Delete or replace it with the one above. Otherwise you'll get a redirect loop.

Step 4 — Wait, verify, push live

  • DNS typically resolves in 1–6 hours; can take up to 24–48 in rare cases.
  • Check progress: dig simenguttormsen.com +short — you should see the four 185.199.10x.153 IPs.
  • Once GitHub shows the green “DNS check successful” in the Pages settings, tick Enforce HTTPS.
  • Visit simenguttormsen.com.

Alternative: Cloudflare Pages

Free, faster CDN, automatic deploys on git push, and you get Cloudflare's free SSL/DDoS protection. Trade-off: you have to transfer DNS (not the domain itself) from GoDaddy to Cloudflare. Worth it long-term for the speed and the cheaper renewal next year.

  1. Sign up at dash.cloudflare.com
  2. Websites → Add a site → enter simenguttormsen.com → choose Free plan
  3. Cloudflare scans your existing GoDaddy DNS, then gives you two nameservers (looks like arnold.ns.cloudflare.com + jane.ns.cloudflare.com)
  4. In GoDaddy: simenguttormsen.com → Nameservers → Change → Enter my own nameservers → paste the two from Cloudflare
  5. Wait for Cloudflare to confirm activation (usually <1 hour)
  6. In Cloudflare: Workers & Pages → Create application → Pages → Connect to Git → pick the GitHub repo → build command empty, output dir /
  7. After first deploy, in the Pages project: Custom domains → add simenguttormsen.com → Cloudflare auto-creates the records

When the domain comes up for renewal next year, you can transfer registration to Cloudflare for ~$10.44/yr (no markup) instead of GoDaddy's $22.99.

If anything breaks

  • 404 after DNS resolves: Pages probably hasn't finished its first build. Wait 5 minutes; Settings → Pages will show a green check when ready.
  • "Improperly configured custom domain" in GitHub Pages: typically means the CNAME file in the repo doesn't match the custom domain. Check the file at the root of the repo contains exactly simenguttormsen.com on a single line.
  • SSL error after DNS resolves: Let's Encrypt provisioning takes ~5–15 minutes after DNS is verified. Don't enforce HTTPS until the lock icon appears.
  • www subdomain doesn't redirect: Double-check the CNAME www → simengut.github.io. record at GoDaddy.
